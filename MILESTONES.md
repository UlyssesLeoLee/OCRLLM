# 项目开发计划与里程碑 (Development Milestones)

根据“基于高质量 OCR 与 PDF 解析的日文小模型训练系统”这一核心目标，以及“高容错、可扩展、多语言”架构原则，制定以下项目落地里程碑。

## 架构核心不变量 (Invariants)
- **绝对的隔离性**：OCR 解析引擎与模型训练业务绝对隔离，两者的桥梁有且仅有 **统一的结构化 JSON 协议**。
- **状态显式管理**：一切 PDF 解析流转状态均由 Rust 主控持久化至 PostgreSQL。
- **不可变数据**：原始 PDF 及其缓存图片只读不写，最终清洗的训练数据为 Versioned Immutable Files。
- **幂等与容灾**：所有解析及清洗任务必须支持重试（Idempotent），防止 OOM 和异常造成的隐性状态损坏。

---

## 🚀 Phase 1: MVP 基础管道建立 (Month 1)
**目标**：跑通首个真实日文/中文 PDF，并稳定输出统一的 JSON 结果。抛弃非关键功能。

*   **Step 1.1**: 定义核心项目结构 (Monorepo) 与 JSON 协议实体定义。
*   **Step 1.2**: 搭建 Rust API 网关与 PostgreSQL 数据库流转引擎（负责任务的 Pending -> Processing -> Done 状态流转）。
*   **Step 1.3**: 搭建 Python 纯文本抽取器（轨道 A），实现 `pdfplumber` 块级文本及坐标解析。
*   **Step 1.4**: 搭建 Python 视觉 OCR 服务（轨道 B），实现 页面转图 -> 版面分析划分段落/标题/表格 -> 调用多语言 OCR 抽取。
*   **Step 1.5**: 对外暴露文档提交 API，包含 `primary_language` 的参数传递。

## 🌍 Phase 2: 多语言清洗与数据工程 (Month 2)
**目标**：治理脏乱差的中日英混排字符，为训练提供绝对干净的“食材”。

*   **Step 2.1**: 在 Rust `data-pipeline` 层实现 **全半角强行收敛机制**。
*   **Step 2.2**: 在 Rust 层实现 **异体中日汉字规范化** 与 **英文字符与汉字间的强制隔离空格** 插入逻辑。
*   **Step 2.3**: 改造 Python OCR 服务，实现基于 `primary_language` 权重的解码策略，修复字形模糊问题。
*   **Step 2.4**: 构建启发式后处理清洗逻辑（消除页眉页脚、拼接跨页段落）。

## 🧠 Phase 3: SFT 监督训练数据车间 (Month 3)
**目标**：将沉淀的统一 JSON 数据转化为大模型能够接纳的 SFT 训练集格式 (JSON Lines)。

*   **Step 3.1**: 实现 **版面结构重建样本生成器**。纯逻辑转换：由 Bounding Box 与类型属性生成结构化文本描述。
*   **Step 3.2**: 实现 **字段抽取任务样本生成器**。利用正则结合 LLM 打标，提取公司名、日期、金额等。
*   **Step 3.3**: （人工介入阶段）引入或开发轻量级 Web 打标工具，对部分抽取失败和 OCR 残缺的内容进行 Human-in-the-loop 修正，保存修正数据为黄金测试集。

## 🤖 Phase 4: 小模型微调与演进 (Month 4-5)
**目标**：选择基底模型，进行领域注入，让模型“看得懂”文档特征。

*   **Step 4.1**: 确定并下载合适的开源基座模型（如 Qwen2.5-7B/14B、Llama-3-8B）。
*   **Step 4.2**: 第一阶段微调：注入 SFT 训练集，教导模型从带有坐标特征的输入中恢复出完美的 Markdown 格式。
*   **Step 4.3**: 领域（LoRA）适配器训练：为不同的表单或文档类型训练专注特征抽取的 Adapter。
*   **Step 4.4**: 基于大语言模型做判定打分器，利用 DPO 或对比学习技术抑制模型的幻觉。

## 🚢 Phase 5: vLLM 推理部署与业务融合 (Month 6)
**目标**：将训练成果投入长期自动化生产和问答服务体系。

*   **Step 5.1**: 利用 AWQ 或 GGUF 等量化技术，降低可用显存体积。
*   **Step 5.2**: 建立 `vLLM`（或其他高效推理引擎）推理后端服务。
*   **Step 5.3**: （后续扩展）将高质量清洗出的 JSON 数据注入向量数据库 Qdrant，启动 Retrieval-Augmented Generation (RAG) 检索问答能力的开发验证。
